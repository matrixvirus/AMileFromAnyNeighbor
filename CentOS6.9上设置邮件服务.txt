2 上手，怎样在两台虚拟机间直接互发邮件,先不管原理
2.1
	有两台虚拟机，系统都是CentOS6.9,它们的IP和域名分别是
		192.168.100.173 chivalry.horseman
		192,168.100.220 pikeman.horseman
	你的DNS服务器的zone文件里还要有这两句
		chivalry.horseman. MX 1 chivalry.horseman.
		pikeman.horseman. MX 1 pikeman.horseman.
	如果你看不懂什么是zone文件，这两句话到底要写到哪里，请专门学习怎么设置dns,
	这里不讲dns。
2.2 在173上编辑/etc/postfix/main.cf
	改几个值
	
	myhostname = chivalry.horseman
	# dns怎么设的myhostname就改成什么
	mydomain = chivalry.horseman
	myorigin $myhostname
	inet_interfaces = all
	inet_protocols = all
	mydestination = $myhostname

	重启postfix
	$ service restart postfix

	在220上进行同样的改变。
2.3 测试
	在173上，安装mailx
	$ yum install mailx
	给220的root用户发一封邮件
	$ mailx root@pikeman.horseman
	先输主题，再输几行话，最后一行一个点号标示结束。

	在220上，假设登录的是root用户，执行
	$ mail
	会看到root用户的邮件,有你刚刚在173上发的那封。

3 mailx命令
	mailx是一个MDA（Mail Delivery Agent) 程序。
	/bin/mail 是/bin/mailx的一个软链。
3.1 MDA是什么
	这段内容来自redhat6文档之部署指导。
	缩写解释:
	  MDA: Mail Delivery Agent
	  MTA: Mail Transport Agent
	MTA调用MDA把进来的邮件放到正确的用户邮箱里。一个MTA把邮件发到本地一个用户
	的邮箱spool文件时也相当于一个MDA。
		打断一下，我不知道怎么翻译spool,字典意思是缠在轴上的线。
		打断一个，我读不懂，放到正确的用户邮箱里，什么用户？linux系统用户吗？
		邮箱是什么，是一个文本文件，还是一个目录，还是一个数据库？
3.2 读邮件
	命令行上，读当前命令行上登录的用户的邮件，直接输mailx，没有参数
	$ mailx
	出现邮件列表，从上到下每封都有一个数字编号
	命令行提示符变成了'& '。后面的命令，以&开头的，都表示在mailx命令内。

	mailx只能读系统用户的邮件，如果要读另一个用户archimedes的邮件，加参数-u
	$ mailx -u archimedes

	读编号为1的邮件,下面几行命令功能一样。
	& type 1
	& print 1
	& show 1
	& 1。


	在命令行文本编辑器vi里读第2封邮件
	& visual 2

	这里有一个概念叫'当前邮件',默认是第1封，你刚刚读了第几封，它就是当前邮件，
	你回复邮件时对的是'当前邮件'。
	查看当前邮件	
	& t
	查看当前邮件的编号	
	& =
	读下一封邮件，下面几行命令功能一样
	& 直接按回车
	& n
	& next

	查看前一封邮件
	& -

	检查邮件列表
	& headers
3.3 发邮件	
3.3.1
	在linux系统命令行上执行mailx直接发邮件,给A@gmail.com和B@gmail.com两个邮箱地
	址发邮件
	$ mailx A@gmail.com B@gmail.com
	然后会要你输主题，正文，
	输完按Ctrl+D或只有一个点号的一行来结束。
3.3.2
	在mailx命令内发邮件
	& mail A@gmail.com

	给当前邮件回复
	& reply
	给指定编号,比如2的邮件回复
	& reply 2

	如果一封邮件有多个收件者，你是其中一个收件者，reply命令回覆给所有人，用R命
	令只回覆发件者	
	只回覆当前邮件的发送者
	& R
	只回覆指定编号，假设2号邮件的发送者
	& R 2
3.4 删除邮件
	删除当前邮件
	& delete
	删除指定编号邮件，比如2号和3号邮件
	& delete 2 3
3.5 
	在mailx内执行linux系统命令
	第一个字母是!,比如执行ls命令
	& !ls

	显示字符，和linux系统的echo命令一样
	& echo 一行话

4.1
	使用POP协议时，客户端下载所有邮件，下载后可以或不删除服务器端的邮件。POP
	协议适合在一个地方读邮件的人，适合没有长时间连网条件的人，不适合网速慢的人
	。
	使用IMAP协议时，邮件在服务器上，客户端可以读，删除，还允许客户端在服务器端
	创造，命名，删除文件夹以更好的组织邮件。IMTP适合在多台机器上访问邮件的人，
	因为在打开前每封邮件只有头信息被下载，适合慢网速。用户可以不用查看，下载
	而删除邮件。
4 cyrus-imapd
4.1 安装
	$ yum install cyrus-imapd
4.2 配置postfix使用cyrus-imapd
	编辑/etc/postfix/main.cf

	# 这下面几行是前面配过的
	myhostname = chivalry.horseman
	# dns怎么设的myhostname就改成什么
	mydomain = chivalry.horseman
	myorigin $myhostname
	inet_interfaces = all
	inet_protocols = all
	mydestination = $myhostname

	# 这下面几行是安装cyrus-imapd后加的
	# 来自不正规文档Mail Server Implementation。
	virtual_mailbox_domains = chivalry.horseman
	# Define the list of virtual mail box addresses.
	# 我检查了，/etc/postfix/vmailbox不存在，可能是要手动创造
	virtual_mailbox_maps = hash:/etc/postfix/vmailbox
	# Use Cyrus Imap as the local delivery agent.
	# /var/lib/imap这个目录cyrus-imapd软件包创造的
	virtual_transport = lmtp:unix:/var/lib/imap/socket/lmtp
	mailbox_transport = lmtp:unix:/var/lib/imap/socket/lmtp
	# 启用TLS.
	smtpd_use_tls = yes
	# 证书和私钥位置
	smtpd_tls_cert_file = /etc/pki/tls/certs/smtp.crt
	smtpd_tls_key_file = /etc/pki/tls/private/smtp.key
	# Tighten rejection criteria slightly.
	# 稍微收紧拒绝标准
	disable_vrfy_command = yes
	smtpd_helo_required = yes
	smtpd_data_restrictions =
	reject_unauth_pipelining,
	permit

5 dovecot
5.1 安装dovecot
	$ yum install dovecot
5.2 配置dovecot
	编辑/etc/dovecot/dovecot.conf，编辑一行
	  protocols = imap pop3 lmtp

	IMAP和POP3协议要求客户端输用户名和密码，默认密码传输没有加密，
	编辑/etc/dovecot/conf.d/10-ssl.conf,编辑一行,它原来是被注释的，取消注释
	  ssl_cipher_list = ALL:!LOW:!SSLv2:!EXP:!aNULL
	这一行使dovecot不使用不安全的SSLv2，SSLv3，

	重启dovecot
	  service dovecot restart
